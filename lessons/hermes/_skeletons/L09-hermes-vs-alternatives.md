## Suggested Pitfalls (for §4)

1. **"Hermes is free, so it's the best"** — free in license, not free in API tokens. Compare total cost of ownership, not sticker price.
2. **"Claude Code is the best because it's the most popular"** — popularity follows the IDE market. Hermes is for a different workload.
3. **Picking Mavis because "beginner-friendly" sounds good** — Mavis is a starting point, not a destination. Plan to graduate.
4. **Switching tools mid-project for marginal gains** — switching cost is high. Pick one and stick for 30 days.
5. **Ignoring the harness-vs-model distinction** — `GGtmmx0MKCI` is explicit: pick by loop, not by model. A great model inside the wrong harness is still the wrong tool.

## Glossary Seeds

- **Coding harness** — an AI agent specialised for writing code (Claude Code, Kilo Code, Cursor's agent mode).
- **Agent harness** — a general-purpose AI agent for non-coding tasks and long-running workflows (Hermes, OpenClaw, Mavis).
- **BYOK (Bring Your Own Key)** — using your own API key instead of a vendor's bundled subscription. Required for Hermes.
- **Max plan** — Claude Code's $200/month tier. Source video says it wins for pure-coding workflows *if you can stomach the cost*.
- **Vertical stack** — when one vendor (like Google) bundles model + harness + sandbox into one product. Compare to Hermes' DIY approach.
