## Suggested Pitfalls (for §4)

1. **Building production code in Hermes.** Use Claude Code for that.
2. **Not setting up project structure first.** Hermes works best with `agents.md` and clear working directories.
3. **Running too many parallel agents.** Hermes sub-agents are slow on cheap models (covered in Hermes L06 / L08).
4. **Forgetting the loop.** Hermes' strength is the iterative verification loop, not one-shot generation.
5. **Not using Skills.** Per the Hermes module, Skills make Hermes dramatically more useful.
