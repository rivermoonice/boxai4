### Step 1 — The four core components

An AI-powered online business in 2026 has four components:

**Component 1: Infrastructure.** Where the business lives.
- Hostinger VPS ($3–$10/mo) for self-hosted applications.
- Hostinger Horizons for no-code builds.
- Domain name ($10–$15/year).
- Email ($5–$10/mo).
Total: $10–$40/mo.

**Component 2: Models.** The AI brain.
- MiniMax API key for daily work ($5–$15/mo at realistic usage).
- Kimi K2.7 base plan for vision/UI work ($10–$20/mo).
- Claude Opus API for hard reasoning (only when needed; can spike).
Total: $15–$50/mo for typical use, can spike higher.

**Component 3: Tools.** The harnesses and connectors.
- Hermes (free, open-source) for agent work.
- Claude Code / Kilo Code (free tier available, BYOK) for coding.
- MCPs from the catalog (free) for connecting to apps.
Total: $0–$20/mo.

**Component 4: Time.** The human investment.
- 90-day launch plan: 10–20 hours/week.
- Ongoing operations: 5–10 hours/week.
- This is the biggest cost by far.

Total monthly cost: **$25–$110/mo at realistic beginner usage.** The "$3/month" headline refers to infrastructure only, not models or time.

### Step 3 — The difference between an AI demo and a real business

**AI demo:**
- Built in 48 hours (per the directory case study).
- Looks polished.
- Has features that work.
- Has zero paying customers.
- Has zero ongoing traffic.

**Real business:**
- Takes 6–18 months to first meaningful income.
- Looks polished eventually.
- Has features customers actually want.
- Has paying customers.
- Has organic traffic.

The distinction matters because beginners conflate "I built a thing" with "I have a business." They're not the same.

### Step 4 — The 90-day launch plan

**Days 1–7: Validate.**
- Pick a niche. List 50 businesses/resources from memory.
- Survey demand. Talk to 20 people in the niche.
- Decide: green light or pick a different project.

**Days 8–14: Build the MVP.**
- Set up infrastructure (Hostinger + domain).
- Set up models (MiniMax API key).
- Set up tools (Hermes + Claude Code).
- Build the directory MVP with 30 listings.

**Days 15–30: Initial traffic.**
- Submit to niche communities.
- Reach out to your network.
- Don't pay for ads yet.

**Days 30–60: Iterate.**
- Add more listings (target 100 by day 60).
- Set up a Hermes Skill for content updates.
- Reach out to listings for paid placements.

**Days 60–90: Monetise.**
- Convert at least 10 listings to paid placements.
- Set up AdSense as secondary.
- Track revenue and conversion.

### Step 5 — When AI replaces you (the warning)

The channel's documented belief is clear: "AI agents are leverage on top of a pre-existing human advantage (network, niche knowledge), not pure automation that replaces the founder."

Translation: **AI doesn't replace you. It amplifies you.**

If you have:
- **A niche advantage** → AI makes you 10× more productive in that niche.
- **A network** → AI helps you serve that network at scale.
- **Real skills** → AI handles the routine, you focus on the judgement calls.

If you have:
- **No niche advantage** → AI doesn't help. You're producing generic content in a crowded space.
- **No network** → AI doesn't help. You have no one to sell to.
- **No skills** → AI doesn't help. You can't evaluate the AI's output.

The "$3/month business" is real, but it works only for founders with prerequisites. AI is leverage on what you already have. It is not a substitute for having anything.

## 4. Common Pitfalls & Troubleshooting

### Pitfall 1 — Confusing the AI demo with a real business

- **Symptom:** You build the directory in 48 hours. You think you have a business.
- **Cause:** "Built" ≠ "running" ≠ "profitable."
- **Fix:** Set a 90-day plan. The build is week 1. The business is weeks 2–90+.

### Pitfall 2 — Spending big on infrastructure

- **Symptom:** You buy premium hosting, multiple models, every MCP.
- **Cause:** Treating a hobby project like a funded startup.
- **Fix:** Cheap stack first. Hostinger + MiniMax + Hermes. Upgrade only when revenue justifies.

### Pitfall 3 — Building before validating

- **Symptom:** You spend a month building. No one visits.
- **Cause:** You built it because AI could, not because there was demand.
- **Fix:** Validate demand before building. Survey 20 people in the niche first.

### Pitfall 4 — Ignoring compliance

- **Symptom:** Your AI-generated content violates copyright or platform ToS.
- **Cause:** Beginners assume AI output is automatically safe.
- **Fix:** Review AI output. Don't reproduce copyrighted content. Comply with platform rules.

### Pitfall 5 — Skipping the legal basics

- **Symptom:** You collect revenue without a business entity.
- **Cause:** "I'll deal with it when I make money."
- **Fix:** Set up a business entity (LLC, sole proprietorship, etc.) before collecting revenue. Pay taxes. Comply with local law.

### Pitfall 6 — Skipping the team

- **Symptom:** You try to do everything yourself.
- **Cause:** "AI can do it all."
- **Fix:** AI amplifies you. It doesn't replace the need for collaboration, networking, or community.

### Pitfall 7 — Measuring the wrong things

- **Symptom:** You track AI tokens used, not paying customers.
- **Cause:** Optimising for the wrong metric.
- **Fix:** Track revenue, conversion, traffic. AI tokens are an implementation detail.
