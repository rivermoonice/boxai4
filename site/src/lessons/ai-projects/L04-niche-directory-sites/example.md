# L04 example — building AI coding tools directory for indie founders

The smallest "do it" demo for [L04 — How to Build a Niche Directory Site with AI in 2026](/lessons/ai-projects/l04-niche-directory-sites/).

Goal: one founder's run of the 50-listings test and viability checklist.
Applied to a real niche: AI coding tools for indie founders.
End with a 90-day plan and the realistic milestones.

## The niche

**AI coding tools for indie founders.**
Covers Cursor, Claude Code, Aider, Continue, Cody, Zed AI, PearAI,
Codeium, Windsurf, Sourcegraph Cody, and 40+ more.
Audience: solo developers shipping SaaS without a team.
Why this niche: founder has shipped two SaaS products using Claude Code
and Cursor; reads AI-coding Twitter daily; has tried 12 tools personally.

## Step 2 — 10 minutes: the 3-competitors survey

Existing directories for AI coding tools:

1. **Awesome-Code-AI** (curated GitHub list) — strong SEO, free
2. **Futurepedia** (broader AI tools, has a code section) — 2M monthly visits
3. **There's An AI For That** (category page, weak curation) — high traffic

Gap: no one curates the *indie founder use case*.
Current directories list everything; none explain what works for a solo SaaS builder.

## Step 4 — the viability checklist

| Check | Result |
|---|---|
| 50 listings from memory? | Yes (50) |
| 10+ people in the niche? | Yes (10) |
| Fewer than 5 existing directories? | Yes (3) |
| Can commit 6–18 months? | Yes |
| Niche supports 200+ listings? | Yes (long-tail on every dev framework) |

**5/5 — start the 90-day plan today.**

## The 90-day plan

### Day 7 — validate and scout

- Finalise niche: "AI coding tools for indie founders."
- Survey existing directories. Gap = indie-founder use case filter.
- One-page PRD: audience is solo SaaS devs; the directory is the only place that ranks tools by "what works for one person shipping a side project."

### Day 14 — MVP on Hostinger Horizons

- Stand up Hostinger Horizons. One site, one template.
- Build the `Listing` entity: name, URL, category, tags, 150-word description, `paid` flag.
- Three page templates: homepage, category (Editor / Agent / Open-Source / IDE-Plugin / Cloud), listing detail.
- Seed 30 listings from the 50-listings list.
- Time spent: about 6 hours total.

### Day 30 — initial traffic

- 30 listings live.
- Submit to: r/SideProject, r/ClaudeCode, r/cursor, IndieHackers, Twitter/X
  thread + tag the 10-contactable list, two Substack newsletters, one Discord server.
- Email the 10 people. Five reply. Two share publicly.
- Live traffic at day 30: roughly 200–400 page views.

### Day 60 — iterate

- Add 50 more listings. Total: 100+ listings live.
- Improve on-page SEO based on which listings get traffic.
- Reach out to listing owners for paid placements.
- First paid placement at day 58: $50/month for a featured slot on Aider's listing.
- Live traffic: 1,500–3,000 monthly page views.

### Day 90 — monetise

- 10 listings on paid placements at $25–$100/month.
- Monthly revenue: $250–$1,000/month.
- AdSense as secondary (still small — ~$30/month at this stage).
- Track traffic, conversion, and revenue weekly.

### Months 6–18 — compound

- SEO compounds. Listings grow. Revenue grows.
- Realistic month-12 outcome: $2K–$5K/month.
- The founder checks the directory twice a month.
- The long-running background-agent loop (see [Hermes L01](/lessons/hermes/l01-what-is-hermes-agent/)) refreshes listings weekly and posts updates to the Discord.

## The viability tier mapping

5/5 checklist pass rate maps to **$5K–$100K/month within 12–18 months**
per the lesson's viability-tiers table.
The realistic outcome for this founder, given the niche advantage and the
specific indie-founder positioning, is **$5K–$10K/month by month 18**
(below the upper bound because the niche is smaller than "all AI tools").

## Cost summary

- Hosting: Hostinger Horizons, $10/month
- Content generation: Kimi K2.7, $15/month (50 listings × 150-word descriptions)
- Domain: $12/year
- **Total to start: ~$25/month**
- **First revenue: day 58 ($50 paid placement)**
- **Break-even on first year: end of month 4**

## What this example proves

The 50-listings test is the real moat test, not a marketing list.
The viability checklist is mechanical: 5/5 maps to a specific outcome band.
The 90-day plan is realistic — first paid placement by day 60, not month 6.
Everything else (stack, cost, timing) is downstream of those three checks.

Apply the same four steps to your niche before you spend a weekend building.
