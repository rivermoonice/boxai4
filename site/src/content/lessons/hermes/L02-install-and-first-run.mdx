In [L01](/lessons/hermes/l01-what-is-hermes-agent/) you decided Hermes is right for you. Now you have to actually install it. Hermes runs in two places — your Mac, or a small rented Linux server called a VPS. Either path takes under twenty minutes if nothing breaks.

## Before you install

The one thing every first-time installer gets wrong is the prerequisite, not the installer — and the installer gives you a misleading error instead of telling you what to do. On a Mac, that prerequisite is **Xcode Command Line Tools**. macOS doesn't ship with build tools by default. Open Terminal (Finder → Applications → Utilities → Terminal) and run:

```bash
xcode-select --install
```

Click **Install** and wait. The first time, the download can take five to ten minutes.

<Callout type="gotcha" title="Don't skip Xcode CLT">
Skip this step and the installer fails with "no developer tools were found" or "command not found." You'll then spend twenty minutes debugging the wrong thing. Install Command Line Tools first, always.
</Callout>

You'll also need a model provider account. Hermes is the harness — it doesn't bundle model access. You bring your own key (BYOK). Any Hermes-supported provider works: Anthropic, OpenAI, DeepSeek, Kimi (Moonshot), MiniMax, or Qwen. Mac path needs macOS 13 or newer.

## Two paths: Mac vs VPS

Before you paste anything, decide where Hermes will live. Getting it right saves you weeks.

<Picture src="/lessons/hermes/L02/mac-vs-vps-decision.svg" alt="Decision diagram: Mac path runs locally and is free but stays awake only while your laptop is open; VPS path costs $2–5/month and runs 24/7." width={880} height={480} formats={["avif", "webp"]} />

**Mac (local).** Hermes is a process on your laptop. Zero monthly cost, fastest iteration, no new skills. Downside: only runs while your laptop is awake — not viable for 24/7 cron jobs. This is the **learning path**.

**VPS (rented Linux server).** Hermes is a process on a small rented server in a data centre. Runs 24/7, isolated from your laptop, $2–$5/month. Downside: small monthly cost, you learn SSH. This is the **production path** — cron jobs, scheduled tasks, anything that has to fire when your laptop lid is closed.

<Callout type="tip" title="Ron's rule">
"If you've never used a terminal before, choose the VPS path. The setup is identical to what a Linux server admin would do, and you'll learn one transferable skill — SSH — that you'll use forever." — Ron, source video `UbK2kXygPUY`
</Callout>

**First-time learners, pick the Mac path.** Faster to iterate, nothing to undo. Once you want a 24/7 setup — cron jobs, automated research, scheduled messages — the VPS path is worth the $3/month.

A handy rule of thumb:

| Situation | Path |
|---|---|
| Brand new to terminals | Mac |
| Want a 7am cron job that runs while you sleep | VPS |
| On a Windows PC | VPS |
| Want the cheapest possible test drive | Mac |
| Plan to connect Telegram or Discord bots | VPS |
| Want config to live forever on one server | VPS |

### The Mac path

Once `xcode-select --install` has finished:

```bash
curl -fsSL https://hermes-agent.newsresearch.com/install.sh | bash
hermes
```

When the wizard offers **quick setup** versus **full setup**, pick **quick** — you can customise later, and on a first install the goal is to confirm the tool runs. The wizard then asks for a provider, your key, and a default model. Skip Telegram and Discord on first install; [L05](/lessons/hermes/l05-mcp-catalog-and-integrations/) wires those properly.

### The VPS path

The VPS path takes longer only because of SSH. Once you're in, the rest is identical to the Mac path.

<Picture src="/lessons/hermes/L02/ssh-vps-flow.svg" alt="Four-step flow: your laptop SSHes into the VPS, the VPS runs the installer, the wizard writes your API key to ~/.hermes/config.yaml, and you chat with Hermes over the same SSH session." width={900} height={280} formats={["avif", "webp"]} />

1. **Rent a server.** Hetzner, DigitalOcean, Vultr, or Linode — Ubuntu 22.04 or 24.04, 2 GB of RAM, $2–$5/month.
2. **SSH in.** `ssh ubuntu@<your-server-ip>`. Type `yes` to trust the host fingerprint, paste the password (the cursor won't move — normal).
3. **Install dependencies.** `sudo apt update && sudo apt install -y git curl`.
4. **Run the installer.** Paste the one-line command. When prompted about importing from OpenClaw, answer **No** — even if you've used OpenClaw before.

<Callout type="gotcha" title="Don't import from OpenClaw">
The source video's creator tried the import and found cron jobs didn't migrate cleanly. A fresh install is more reliable than a migrated one, even if it means rebuilding a couple of skills.
</Callout>

## Pick your model

Once the installer finishes, the wizard wants a model provider and an API key. Picking the right model is the difference between a $5/month learning habit and a $50/month surprise. Keep it boring for now — you can switch providers later by re-running `hermes setup`.

| Provider | Pick when… |
|---|---|
| Kimi K2.6 (Moonshot) | Cost matters most; daily use, non-English |
| Claude Sonnet 4.8 (Anthropic) | Reliable quality, broad capability |
| Claude Opus 4.8 (Anthropic) | Hard reasoning, important deliverables |
| DeepSeek V4 Pro | Strong tool-call reasoning, low cost |
| OpenAI GPT-5.x | Solid general-purpose |
| MiniMax M3 | Cost-efficient execution |

For 50–100 messages a day at the cheap tier, expect **$5–$10/month** in API usage. Claude Opus runs roughly 4–5× that.

**Set a hard monthly budget cap** in the provider's billing dashboard on day one. Without it, a runaway session at 3am can burn real money in minutes. $20 is a reasonable beginner ceiling. Then paste the key into the wizard — and never paste it anywhere else. API keys are passwords.

<Callout type="bottom-line" title="Ron's bottom line: pick cheap, upgrade later">
Start with Kimi K2.6 or DeepSeek V4 Pro. Once the loop is working and you can see what Hermes actually does, upgrade to Claude Sonnet only for the queries that need it. Pay-for-quality is a great habit; pay-for-everything is a fast way to overspend.
</Callout>

## Verify it works

You have a working install when all three of these return without error:

```bash
hermes doctor          # checks Python, SSL, dependencies
hermes status          # shows config, API keys, version
hermes skills list     # enumerates available skills
```

The final smoke test: launch the chat and type `hi`. If Hermes responds, the install is complete.

```bash
hermes
> hi
< Hello! How can I help today?
```

If you see an error, the most common cause is a wrong API key in the wrong provider field, or a missing `curl`/`git` on the VPS path. Re-run `hermes setup` or install the dependencies and retry.

### The exercise

In the terminal where Hermes is installed, confirm these three commands all return without error:

```bash
hermes doctor
hermes status
hermes skills list | head
```

If everything is green and you can see your provider listed in `status`, your install is verified.
