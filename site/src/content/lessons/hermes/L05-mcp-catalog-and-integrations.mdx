By now you've installed Hermes, sent a few messages, and started to think of it as a daily driver. The next question is: **how do I get it to do things in the apps I already use?** Linear, GitHub, Notion, Discord — the work that actually lives somewhere. That's what **MCP** is for, and this lesson is the four commands you need to know.

This is lesson five in the [Hermes Agent track](/lessons/hermes/). If you haven't installed Hermes yet, start with [L02 — Install and First Run](/lessons/hermes/l02-install-and-first-run/). For the harness concept first, see [L01 — What Is Hermes Agent](/lessons/hermes/l01-what-is-hermes-agent/).

## The hook

Here's the thing an AI assistant can do today that a chatbot from 2023 could not: **read your Linear backlog, find the issue assigned to you this week, draft a reply, and post it back — all from a single sentence in the chat.** The piece of software that makes that possible is an **MCP server**, and the standard that lets your agent talk to one is the **Model Context Protocol**.

The reason this is a lesson and not a footnote: the *safe* way to do this changed in May 2026. Before the catalog, you had to find a community MCP server, copy a JSON snippet into your config, manage an API key by hand, and hope the code wasn't malicious. **The Hermes team now ships a built-in catalog of pre-reviewed MCPs.** For a beginner, the catalog is the only on-ramp that exists; for everyone else, it's the sane default.

## The mental model

There are two ways to give Hermes a new tool. Pick the right one for the situation.

<Picture src="/lessons/hermes/L05/catalog-vs-custom-mcp.svg" alt="Diagram: catalog MCP path versus custom MCP path" width={900} height={480} formats={["avif", "webp"]} />

**Catalog MCP** is the safe path. The Hermes team has read the source code, tested the install, and signed off on every entry. You run one command, click through an OAuth prompt, review a tool whitelist, and you're done. As of June 2026, the catalog ships **Linear** and **n8n**. That list will grow — run `hermes update` and `hermes mcp` to see what's available today.

**Custom MCP** is the flexible path. The app you need isn't in the catalog, so you go to GitHub, find a community MCP someone wrote, copy its `command` and `env` into your `mcp.yaml`, add your own API key, and run it. Higher ceiling, lower floor. Some community MCPs are excellent; some are malicious. The catalog exists because the gap between those two is exactly where new users get burned.

<Callout type="gotcha" title="The rule">
**First MCP install: catalog.** Don't graduate to custom until you've run a catalog install end-to-end and you understand where the credentials live and how to revoke them. L07 — Security Essentials — covers the threat model in detail.
</Callout>

### The four commands

The whole catalog workflow is four commands. This is the part worth memorising — every other piece of MCP knowledge hangs off these.

| Command | What it does |
|---|---|
| `hermes update` | Pulls the latest Hermes release, including the latest catalog manifests. Run this first. |
| `hermes mcp` | Shows the catalog. Lists every approved MCP server with a short description. |
| `hermes mcp install <name>` | Installs a specific MCP from the catalog. Prompts for credentials and shows the tool whitelist. |
| `hermes mcp configure <name>` | Re-opens the tool whitelist for an installed MCP. Use this to tighten permissions later. |

You can see them on screen by running `hermes mcp --help`. The output is short and worth a glance before your first install — the subcommand list tells you which verbs the catalog actually exposes:

```
usage: hermes mcp [-h] [--accept-hooks]
       {serve,add,remove,rm,list,ls,test,configure,config,login,reauth,
        picker,catalog,install} ...

Manage MCP server connections and run Hermes as an MCP server.
MCP servers provide additional tools via the Model Context Protocol.
Use 'hermes mcp add' to connect to a new server, or 'hermes mcp serve'
to expose Hermes conversations over MCP.
```

`picker` and `install` are the two you'll touch. `configure` is the one you'll come back to. The rest are there when you need them.

## Pick your tool

Which app should you connect first? The honest answer is: **the one that already holds work you'd do this week.** Here are the realistic first picks, in order of how cleanly they install and how immediately useful they are.

**Linear** — the cleanest first install, and the recommended one for this lesson. It uses OAuth, so there's no API key to copy/paste. The catalog install gets you `find_issues`, `create_issue`, and a few other read/write tools in about three minutes. Pick this if your team plans work in Linear.

**n8n** — pick this if you already run n8n workflows and want Hermes to inspect, trigger, or edit them. Uses an API key instead of OAuth.

**GitHub, Notion, Discord, Telegram** — not in the catalog as of June 2026. The community has MCPs for all of them, and the quality varies. Do not install these as your first MCP.

| App | Path | Effort | When to pick it |
|---|---|---|---|
| Linear | Catalog | 3 min | First install, OAuth, immediate payoff |
| n8n | Catalog | 5 min | You already run n8n workflows |
| GitHub | Custom | 15–30 min | Read source first; community MCPs vary |
| Notion | Custom | 15–30 min | Same caveat as GitHub |
| Discord | Custom | 15–30 min | Use a vetted MCP; never paste a bot token into a random repo |
| Telegram | Custom | 15–30 min | Same caveat as Discord |

The decision tree is the same one the source video draws: **catalog first, custom second, write-your-own last.** Beginners stop at the catalog until they're confident.

### What an MCP actually is (the short version)

An **MCP** is a small server program that speaks one app's API. Each MCP exposes a list of **tools** — discrete functions like `find_issues` or `create_issue` — that the agent can call. The MCP is the bridge between the model in Hermes and the app on the other side. One MCP per app. The agent routes tool calls to the right MCP by name.

That's the whole model. The rest is configuration.

## Try it

### Install Linear from the catalog

Linear is the cleanest first install because it uses OAuth. No API key, no `.env` editing, no copy/paste of long strings.

**1. Update.** Run `hermes update` to pull the latest catalog manifest.

**2. Browse.** Run `hermes mcp`. You should see `linear` and `n8n` listed with short descriptions.

**3. Install.** Run `hermes mcp install linear`. Hermes will:
- Show you a link to authenticate with Linear.
- Wait while you click the link, log in to your workspace, and grant access.
- Return you to a **tool whitelist** screen.

**4. Whitelist.** By default, read-only and safe-write tools are checked; destructive tools are unchecked. **Read the list.** If you wouldn't give a new intern the `delete_workspace` permission, don't give it to an MCP. Hit enter to save.

**5. Use it.** From the Hermes chat:

> *"Find all issues assigned to me in Linear this week and summarise them in three bullets."*

The agent will route the call to the Linear MCP and return real results from your workspace. If the call fails, run `hermes mcp` again — your OAuth token may have expired, and `hermes mcp configure linear` will let you re-authenticate.

**6. Tighten later.** When you come back to this MCP a week from now and want to revoke a permission, run `hermes mcp configure linear`. This is the single most important defensive habit: re-review the whitelist on every MCP you have installed.

### Where the credentials live

After install, your Linear OAuth token is cached at `~/.hermes/mcp-tokens/linear.json`. If you installed an API-key MCP (like n8n), the key is written to `~/.hermes/.env`. Both are plaintext on your disk. Treat them like passwords: don't commit them, don't paste them into chats, and clean up the files for any MCP you've uninstalled. L07 — Security Essentials — covers this in depth; the only thing you need from this lesson is the *locations*, not the full threat model.
