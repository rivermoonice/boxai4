If you've ever asked ChatGPT a question, watched it give you a confident answer, and then realised the answer was completely wrong — you already know the gap between **"an AI can chat"** and **"an AI can do real work."** That gap is exactly what an **AI agent harness** exists to close. Hermes Agent is one of the most popular open-source harnesses in 2026, and the one we'll use throughout this course.

This is the first lesson in the academy. **No install yet** — this is pure concept. The install lesson is next, in [L02 — How to Install Hermes Agent for Beginners](/lessons/hermes/l02-install-and-first-run/).

## The hook

Pick a task you do every week. **"Summarise today's top 3 AI news stories and post them to my Discord at 7am."** Could a chatbot do that? No. A chatbot waits for you to open a tab, type a question, and copy-paste the answer. The 7am thing is a background job. That's the gap.

A **harness** turns an AI from a thing you chat with into a thing that *works for you in the background* — reading files, running commands, scheduling jobs, talking to other apps, and remembering what it learned last week.

## The mental model

The single most important idea in this course: **a model is a brain, a harness is a body**.

<Picture src="/lessons/hermes/L01/brain-vs-harness.svg" alt="Diagram: model is the brain, harness is the body" width={800} height={420} formats={["avif", "webp"]} />

A **model** (Claude, GPT, DeepSeek, Kimi, Qwen) takes text in and returns text out. On its own, it can't read a file on your computer, can't run a command, can't remember what you said yesterday. The model is the *brain*.

A **harness** is the program that wraps the model and gives it those abilities. The harness is what lets the AI *do* work in the world: read your codebase, execute shell commands, post to GitHub, schedule a task for 6am tomorrow, or remember a preference across sessions. The harness is the *body*.

<Callout type="tip" title="Ron on the bottleneck">
"So, how I've been approaching building stuff for AI has been sort of wrong or rather inaccurate. It's not a model issue, but rather a harness issue." — Ron, source video `GGtmmx0MKCI`
</Callout>

**If you take away one thing from this lesson, take this: when your AI messes up, the problem is almost always the harness, not the model.** In 2026 the models are good enough. The differentiator is what wraps them.

## Pick your tool

In 2026, harnesses split into two broad categories. The decision is one of the first things you'll make, and getting it right saves you weeks.

**Coding harnesses** are built around a codebase. Claude Code, Kilo Code, and the agent mode in Cursor live here. They start from files, repositories, pull requests, IDEs, and CI pipelines. They run the actual code and check the test output. They are designed to be the *engineering shell* of an agent.

**Agent harnesses** are built around a runtime. Hermes and OpenClaw live here. They start from sessions, tools, memory, scheduled jobs, channels (Discord, Telegram, Notion), and routing rules. They are designed to be the *autonomous shell* of an agent — persistence across sessions, multi-channel operation, gradual improvement as you use them more.

<Callout type="tip" title="The shorthand">
Coding harnesses verify by **running**. Agent harnesses verify by **inferring**.
</Callout>

### When to pick which

| Goal | Pick | Why |
|---|---|---|
| Build a full-stack app from scratch | Claude Code or Kilo Code | Coding harnesses verify by running tests; the work is inside a codebase. |
| Refactor an existing codebase | Claude Code | It owns the engineering shell and respects your Git workflow. |
| Post a 7am news summary to Discord | Hermes | Long-running, persistent, runs without you. |
| One-off code question | Claude Code | Single-session deliverable; no need for persistence. |
| Multi-day research project with notes that survive | Hermes | Memory across sessions is the differentiator. |
| A daily-driver helper that learns your style | Hermes | Gets more capable the longer it runs. |

**Rule of thumb:** if your work is mostly inside a codebase — building, refactoring, testing — open a coding harness. If your work is mostly outside a codebase — research, scheduling, multi-channel automation, long-running personal tasks — open Hermes. If you do both, run both. They don't conflict.

<Callout type="gotcha" title="Beginner trap">
Beginners sometimes pick Hermes because it's free and open-source and assume it'll do everything. It won't. Hermes is for the *second* category. For shipping software, use a coding harness even if you have to pay for it.
</Callout>

For a deeper comparison once you've tried both, see [L09 — Hermes Agent vs Claude Code vs Mavis](/lessons/hermes/l09-hermes-vs-alternatives/).

### What Hermes specifically is

Hermes is an open-source agent harness that runs on your laptop or on a small rented server (a VPS). It is one of the most-starred open-source AI projects on GitHub in 2026, with over 200,000 stars. It is not a model — it is the program that *wraps* a model.

From the project's documentation: *"We are not a coding co-pilot tethered to an IDE. We are an autonomous agent that gets more capable the longer it runs."*

In practice, that means Hermes can:

- Read and write files in a working directory you specify.
- Run shell commands and verify their output.
- Remember context across sessions, especially things you flag as important.
- Be scheduled to run tasks at specific times (covered in [L08 — Automation: cron, kanban, sub-agents](/lessons/hermes/l08-automation-cron-kanban-sub-agents/)).
- Connect to other apps like GitHub, Linear, Notion, and Discord via MCP (covered in [L05 — MCP Catalog and Integrations](/lessons/hermes/l05-mcp-catalog-and-integrations/)).

What Hermes is **not**:

- **Not a chat app.** It's a terminal program. You can wrap it in a desktop UI, but the underlying interface is text.
- **Not a model.** You bring your own model API key (BYOK) — Anthropic, OpenAI, DeepSeek, Kimi, etc. Cost is whatever you spend on the model.
- **Not magic.** It improves with use and feedback, but only if you put in the time. The source video puts it bluntly: *"you'd have to imagine you have to spend hundreds or thousands of hours if you have no technical experience on training up your Hermes agent to function as well as if you could put the same model in a coding harness."*
