## Suggested Pitfalls (for §4)

1. **Vibe coding production code.** Per the source: "Vibe coding with intent-free prompts burns tokens and breaks on drag-handlers."
2. **Real coding a prototype.** Real coding is overkill for throwaway work.
3. **Ignoring the cost.** Vibe coding on Claude Max can burn $200 in an afternoon.
4. **Skipping the review step.** Vibe coding needs a code review before shipping.
5. **Mixing them without a plan.** Pick one or the other per project.

## Glossary Seeds

- **Vibe coding** — using AI to generate code from natural-language prompts without writing it yourself.
- **Real coding** — using AI as a tool while you write and verify the code yourself.
- **Loop** — when the agent retries a task until a success check passes.
- **PRD (Product Requirements Document)** — a written description of what you're building, used to anchor vibe coding prompts.

---

## Notes for the Drafter

- **Channel-voice alignment.** The source video's main point: vibe coding works for prototypes and exploration. Real coding is needed for production. Both have a place.
- **The PRD step is critical.** Per the source: "Draw a PRD, pick a model per task, and review code before shipping."
- **Frame as a spectrum, not a binary.** Many beginners land in the middle (vibe coding for exploration, then real coding for the production version).
